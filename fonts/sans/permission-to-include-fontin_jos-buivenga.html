Op 24 jun. 2020, om 13:41 heeft Alex Povel <mail@alexpovel.de> het volgende geschreven:

Dear ladies and gentlemen,

on the hunt for a suitable sans-font, I found Fontin Sans on your website here: https://www.exljbris.com/fontinsans.html.

Now, that font is free to download and use, which is excellent. I planned on using it for a personal PDF (thesis) in LaTeX (a typesetting system). A closer look at the exact license reveals that it may be embedded into PDF, which works out fine.

However, the font may not be distributed. Now, the thesis as a LaTeX project is supposed to be under version control using git. The resulting "repository" is then supposed to be published (so that both the resulting PDF but also the source code for producing it is public, which is great in the spirit of open science).

Taking the example of Fira Code, a GitHub repository for a git project consisting of a font looks like:

https://github.com/tonsky/FiraCode

A thesis repository looks slightly different of course (the font is only a sub-part of it, and there is a lot more other stuff, unrelated to fonts).

As such, everyone can access the project and download it. If that project "ships" with the font, it technically also distributes it: everyone downloading the project would download the font files lying somewhere in it.

The project would be in a broken state if it included all files required for producing the PDF *except* for Fontin Sans. If it still specified using Fontin Sans in the source code, but without the corresponding files present, no PDF can be produced. Essentially then, Fontin Sans couldn't be used altogether. Fontin Sans trumps other similarly liberal alternatives, so that would be a bummer.

In the license it states to ask for explicit permission for such cases. This is what I hereby do. May the Fontin Sans font files be included in a publicly available git repository (alongside their license file, attributing you)? Additional attribution could be given in the produced PDF, for example.

Regards,

Alex Povel
